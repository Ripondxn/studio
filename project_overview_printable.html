Project Overview: Business Management System

1. Project Overview & Architectural Design

This is a modern, web-based Enterprise Resource Planning (ERP) system designed to manage a wide range of business operations. At its core, it helps a company manage everything from properties and finances to employees and inventory. A key feature is its AI-powered document analysis, which can automatically extract data from invoices and other documents, saving significant manual effort.

Architectural Design:

  • Framework: The project is built using Next.js, a popular React framework. This is evident from the .tsx file extensions and the src/app directory structure.
  • Language: The entire codebase is written in TypeScript, which adds strong type-checking to JavaScript, reducing bugs and improving code quality.
  • Backend & AI: The backend logic and AI features are handled by Genkit, an open-source framework connecting to Google's Gemini Pro Vision model to perform tasks like document analysis.
  • Authentication & Database: The system uses Firebase for user authentication and likely a cloud database like Firestore or Cloud SQL.
  • Frontend (UI): The user interface is a Single Page Application (SPA) built with React components, providing a fast and responsive user experience.

2. User Roles & Permissions

The system uses a flexible and secure Role-Based Access Control (RBAC) model. Instead of assigning permissions to each individual user, permissions are assigned to a Role, and users are assigned one or more roles. The UI is built dynamically based on the logged-in user's role.

Defined User Roles (Examples):

  • Admin: Has full access to all modules and can manage user roles and permissions in the "Access Control" settings.
  • Accountant: Has access to the Finance, Vendor, and Customer modules. Can view all financial data but may only have edit permissions for certain actions.
  • User (e.g., Property Manager): Might only have access to the Lease and Tenant modules to manage properties and tenant information. They cannot see financial or HR data.

3. Dashboard Layout & UX/UI

The application uses a classic and user-friendly dashboard layout.

  • Side Navigation Bar: A persistent vertical menu on the left for primary navigation.
  • Content Area: The main area to the right of the navigation where all forms, tables, and data are displayed.
  • Dynamic Menus: The items in the side navigation are dynamically generated based on user roles, decluttering the interface.
  • Forms & Dialogs: Data entry is handled through clear forms and pop-up dialogs to keep the user in the context of their current page.
