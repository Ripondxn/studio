Master Prompt: Replicating the Business Management System

Project Title: Comprehensive Business Management System (ERP)

High-Level Goal: Develop a full-stack, enterprise-grade web application to serve as a comprehensive Business Management System. The application must be scalable, secure, and feature-rich, with a modern, responsive user interface. The primary goal is to create a fully functional prototype with a complete codebase, including a key feature for AI-powered data extraction from documents.

Core Technologies:

  • Framework: Next.js (with App Router)
  • Language: TypeScript
  • Styling: Tailwind CSS
  • UI Components: Shadcn/UI (or a similar modern component library)
  • Authentication: Firebase Authentication (Email/Password and Google Sign-In)
  • Database: Firestore (for all application data)
  • AI Backend: Genkit (running on a serverless environment)
  • AI Model: Google's Gemini Pro Vision

Part 1: Foundational Setup & Architecture

  1. Initialize Next.js Project: Create a new Next.js project with TypeScript and Tailwind CSS. Set up the src/ directory structure.
  2. Firebase Integration:
    • Create a src/lib/firebase/config.ts file to initialize the Firebase app. Use environment variables for all Firebase configuration keys (NEXT_PUBLIC_FIREBASE_*).
    • Set up Firestore rules to be secure by default (deny all reads/writes), which will be opened up on a per-collection basis later.
  3. UI & Layout:
    • Integrate Shadcn/UI by initializing it in the project.
    • Create a primary layout in src/app/admin/layout.tsx. This layout must include a persistent, collapsible Side Navigation Bar, a Header Bar, and a main Content Area.
    • The navigation items must be dynamically generated based on the user's role.

Part 2: Authentication & Role-Based Access Control (RBAC)

  1. Authentication Logic: Implement user sign-up, sign-in, and sign-out using Firebase Authentication. Use React Context or a state management library to provide authentication status.
  2. RBAC Data Model: In Firestore, create two collections: users (stores roles array) and roles (stores permissions map).
  3. Middleware for Route Protection: Create src/middleware.ts to protect all routes under /admin.
  4. "Access Control" Admin Module: Create a page at /admin/access-control (visible only to 'admin' role) to manage users, roles, and permissions.

Part 3: AI-Powered Document Processor (Genkit & Gemini)

  1. Genkit Setup: In src/ai, initialize Genkit and configure it to use the Google AI plugin with the Gemini Pro Vision model, sourcing the API key from environment variables.
  2. Document Processing Flow:
    • Create src/ai/flows/process-document-flow.ts.
    • Define Zod schemas: ProcessDocumentInputSchema for the base64 URI and ProcessedDocumentSchema for the structured output (vendor, date, items, total, etc.).
    • Create a Genkit Prompt (documentProcessorPrompt) engineered to be precise, and wrap it in a Genkit Flow (processDocumentFlow).
  3. Frontend UI for Document Upload: Create a component for file upload that converts the file to a data URI, calls the Genkit flow, and displays the returned JSON data.

Part 4: Core Module Implementation

For each module listed below, implement the full CRUD (Create, Read, Update, Delete) functionality, respecting the RBAC permissions at every step.

Instructions for each module:

  • Data Schema: Define the Firestore data structure.
  • UI (Table View): Use Shadcn/UI's Table component for a list view with sorting, filtering, and pagination.
  • UI (Forms & Dialogs): Use Shadcn/UI's Dialog for creating/editing records. Use Zod for validation.
  • API Endpoints: Use Next.js Server Actions or API routes for database operations.
  • RBAC Integration: Conditionally render UI and protect actions based on user permissions.

Modules to Implement:

  • Finance (Invoices, Bills, Payments)
  • Lease (Properties, Landlords, Leases)
  • Tenant (Tenant Directory, Contracts)
  • Vendor (Vendor List, Vendor Bills)
  • Human Resource (Employees, Leave, Compensation)
  • Project Management (Projects, Tasks)
  • Stores / Inventory (Products, Stock Locations, Purchase Orders)
  • Specialized Modules (Optional): Rent-A-Car, Car Sales

Final Instruction:

Begin with Part 1 and confirm completion before moving to the next part. Work iteratively and generate code that is clean, well-commented, and follows modern TypeScript/React best practices. The final output should be a complete, runnable project that can be cloned from a repository and run with minimal setup.
